# Sudoku Engine v1.0.0 Release Notes

## Summary

`v1.0.0` is the first stable release of Sudoku Engine. It packages the Spring
Boot API, static browser UI, solver visualization, Docker runtime, CI pipeline,
and project documentation as one deployable application.

## Highlights

- Generate Sudoku puzzles by `EASY`, `MEDIUM`, `HARD`, or `EXPERT` difficulty.
- Solve boards with either plain backtracking or MRV, including solver metrics.
- Validate row, column, and box violations through the API and live UI feedback.
- Request safe hints with cell, value, and explanation.
- Play in a responsive browser UI with keyboard navigation, timer, reset,
  pause, pencil notes, hints, and completion verification.
- Visualize solver steps with play, pause, reset, speed control, and metrics.
- Browse API documentation through Swagger UI.
- Run as a packaged JAR, Docker container, or Docker Compose service.

## Release Artifacts

- JAR: `backend/target/sudoku-engine-1.0.0.jar`
- Docker image: `sudoku-engine:v1.0.0`
- Health endpoint: `/actuator/health`

## Verification

The release candidate was verified locally with:

```bash
mvn -B clean verify
docker build -t sudoku-engine:v1.0.0 .
docker run --rm -p 8080:8080 -e PORT=8080 -e SPRING_PROFILES_ACTIVE=prod sudoku-engine:v1.0.0
```

## Known Issues

- The public Render URL must be added after the hosted service is created.
- GitHub Release publishing requires authenticated GitHub access. If GitHub CLI
  is unavailable, create the release from the GitHub web UI and upload the JAR.
- Selenium may log a Chrome DevTools Protocol version warning with newer Chrome
  versions. The headless browser tests still pass.
- The application is stateless in v1.0.0; games and preferences are not saved
  across browser refreshes.
